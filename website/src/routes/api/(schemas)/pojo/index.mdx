# pojo

Creates a POJO schema.

```ts
const Schema = v.pojo<TMessage>(message);
```

## Generics

- `TMessage` <Property {...properties.TMessage} />

## Parameters

- `message` <Property {...properties.message} />

### Explanation

With `pojo` you can validate that the input is a plain JavaScript object. If the input is not a plain object, you can use `message` to customize the error message.

> Unlike <Link href="../object/">`object`</Link> and <Link href="../record/">`record`</Link>, this schema only checks the root type and does not validate entries. It also accepts plain objects with a `null` prototype.

## Returns

- `Schema` <Property {...properties.Schema} />

## Examples

The following examples show how `pojo` can be used. Please see the <Link href="/guides/objects/">object guide</Link> for more examples and explanations.

### Plain object schema

Schema to validate any plain object.

```ts
const PlainObjectSchema = v.pojo();
```

### Custom message

POJO schema with a custom error message.

```ts
const PlainObjectSchema = v.pojo('A plain object is required.');
```
